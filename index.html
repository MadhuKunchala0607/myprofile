Expence Tracker

This project is an expense tracker that allows users to add and categorize their expenses into food, transport, and medical categories. Users input the title, amount, date, and category of each expense, which is dynamically displayed on the webpage. Each expense can also be deleted, and the total sum is updated accordingly. The project includes a pie chart that visually represents the proportion of each category using a conic-gradient. The chart updates automatically as expenses are added or removed. This project enhances the understanding of DOM manipulation, event handling, and CSS for data visualization in JavaScript.

GitHub Repository See Live

Digital Clock

I built a simple digital clock that dynamically updates the current time and date using JavaScript. The clock displays time in the format hours:minutes:seconds and updates every second. Additionally, it shows the current date in a human-readable format, such as "Tue Oct 2 2024." This project leverages JavaScript's Date() object for fetching real-time data and uses setInterval() to refresh the time every second. The project showcases my ability to work with JavaScript DOM manipulation, event-driven updates, and creating functional, interactive user interfaces.

GitHub Repository See Live

E-commerce Product Display with Cart Functionality

I developed an e-commerce project that fetches product data from the Fake Store API and displays it as interactive product cards on the webpage. Each card includes the product's title, image, and price, along with an "Add to Cart" button. When users click the button, the product is added to a shopping cart, and the total price is updated accordingly. The project features local storage functionality, allowing users to retain their cart items even after refreshing the page. Each added product's details are stored in the browser's local storage, enabling persistent access to the cart data. This project demonstrates my skills in asynchronous JavaScript (using async/await for API calls), DOM manipulation for dynamic content generation, and local storage for data persistence.
